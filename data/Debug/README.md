# 边界与异常测试

## 四、Debug — 边界与异常测试

### 4.1 empty

| 属性 | 值 |
|------|-----|
| V / E | 0 / 0 |
| 测试点 | 空 CSV 文件加载后，所有 22 个命令在空图上的行为 |

**特殊性**：`places.csv` 和 `roads.csv` 均为空文件（0 字节）。LOAD 成功创建空图后，检验每个命令在"图为空"的极端情况下的输出是否正确

**构造来源**：设计者不小心传了空文件，或者想在空文件基础上构造

**能卡住的实现**：任何在调用前不检验存在性的算法

**如何处理**：严格处理算法逻辑，增强鲁棒性


### 4.2 trivial

| 属性 | 值 |
|------|-----|
| V / E | 87 / 0 |
| 测试点 | 87 个孤立节点（ECNU 真实地点数据，无道路） |

**特殊性**：有顶点但无边——所有节点度数为 0 ，检验每个命令在平凡图的极端情况下的输出是否正确

**构造来源**：设计者只知道地点信息，要根据地点来进一步构建道路

**能卡住的实现**：任何在调用前不检验存在性的算法

**如何处理**：严格处理算法逻辑，增强鲁棒性

### 4.3 illegal_command

| 属性 | 值 |
|------|-----|
| V / E | 87 / 26（ECNU 真实数据） |
| 测试点 | 非法命令名、超长字符串、空行、特殊字符的容错 |


**特殊性**：输入非法命令大全：错误文件路径、`WRONG_COMMAND`、中文乱码、256+ 字符的缓冲区压制、纯空行、`\o \n \r ^D` 字符等。检验 `ProcessCommand` 对未知指令的统一处理（`CommandProcessor::ProcessCommand:未知指令`）以及 LOAD 对不存在文件的错误报告。

**构造来源**：使用者不小心输错了命令

**能卡住的实现**：任何在调用时不严格检验输入的算法

**如何处理**：严格处理算法逻辑，增强鲁棒性

### 4.4 illegal_data

| 属性 | 值 |
|------|-----|
| V / E | 6 / 7 |
| 测试点 | CSV 中包含非法字段值——负数距离/时间 |


**特殊性**：7 条道路中 2 条非法——`P0004→P0005` 的 `walk_time = -5`、`P0001→P0006` 的 `distance = -50`。预期 LOAD 时 `ValidCheck` 抛出异常，整批数据拒绝加载，图保持为空。

**构造来源**：使用者不小心输错了数据

**能卡住的实现**：Dijkstra最短路径算法

**如何处理**：直接在输入层拒绝错误数据，把风险扼杀在摇篮中；其他处理包含忽略负权边或在算法执行前做一次遍历，或者在图中维护一个HasNegative用来记录是否含有负权边，但考虑到真实性，在遇到负数权重后，会直接return；不过值得一提的是，在web版中，负权边可以加载，但会被标亮，且严格禁用任何算法

**重要发现**：此数据集直接导致发现了 `InsertEdge` 中 `ValidCheck` 从未生效的 bug——`operator[]` 会就地创建默认 `EdgeNode{0,0,"open"}`，校验对象张冠李戴，具体请见实验报告.md中介绍

## 五、设计方法论

### 5.1 对抗样例设计原则

| 原则 | 对应数据集 | 说明 |
|------|-----------|------|
| **场景还原** | shanghai_metro, knight_graph | 分别对应现实与虚拟的拓扑结构 |
| **放大复杂度** | complete_graph_K100, prim_killer_K2000 | 用稠密图暴露 O(V²) 和 O(E log V) 的差异 |
| **边界压力** | mass_grid_K1500 | 中等规模（1500 节点），测试常系数和内存 |
| **非法输入** | illegal_data, illegal_command| 测试防御性编程 |
| **极端边界** | empty, trivial | 空图和全孤立图——每个命令的空行为 |

### 5.2 创新性

1. **非法数据注入**：直接在 CSV 嵌入非法值，测试健壮性。直接导致 ValidCheck bug 的发现

2. **性能阶梯设计**：为了明显卡住Prim算法，从K600->K1200->K2000三级规模递进

3. **成对对比数据**：同数据、异命令，在 testall 中直观展示 O(V²) vs O(V+E) 的差异

4. **创新建模**：上海地铁展示此项目在实际规划中的作用，骑士图可以用作算法研究

### 5.3 运行时间

> 为加快对抗速度，提升了CPU性能，实际数据取决于电脑参数
> 采用O2优化，O3优化经测试可以运行且速度明显更快，但未使用
