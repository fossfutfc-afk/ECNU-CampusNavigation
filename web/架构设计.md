# CampusNavigation 图形化界面 — 架构设计 v2

> 拓展任务 3 · 方案 A：Flask 中间层 + Leaflet.js 前端

---

## 一、运行流程

```
浏览器 (HTML+CSS+JS)  ←fetch→  bridge.py (Flask)  ←stdin/stdout→  CampusNavigation.exe
     Leaflet 地图               HTTP API                subprocess      (不修改)
```

1. 启动 `bridge.py`，`CampusNavigation.exe` 作为长期子进程保持运行
2. 浏览器打开 `localhost:5000`
3. 点击按钮 → JS 发 fetch → Flask 向 exe 发送 CLI 命令 → 解析输出返回 JSON → 前端更新地图

## 三、API 设计

| 路由 | 方法 | 说明 |
|------|------|------|
| `/` | GET | 返回 index.html |
| `/api/load` | POST | `{file}` → 加载指定数据目录 |
| `/api/graph` | GET | 返回全部顶点 + 边（坐标由 bridge 自动网格布局） |
| `/api/shortest` | POST | `{from, to, mode}` → 最短路径 |
| `/api/mst` | POST | MST 边集合 |
| `/api/critical` | POST | 关键节点 + 关键边（朴素） |
| `/api/testall` | POST | 运行 testall.exe，返回结果 |

## 四、前端设计

### 4.1 页面布局

```
┌──────────────────────────────────────────────────────┐
│  🏫 CampusNavigation 校园导航系统                      │
├────────────────────────────────┬─────────────────────┤
│                                │   📍 数据           │
│        Leaflet 地图             │   [ECNU样例 ▾]    │
│        (占 70% 宽度)            │   [加载]           │
│                                │                     │
│   ● = 节点（按类别着色）        │   🗺 算法           │
│   ─ = open 边 (绿)             │  起点: [点击地图]  │
│   - - = closed 边 (红)         │  终点: [点击地图]  │
│   ═══ = 最短路径 (蓝粗)        │  [最短路径 DIST]   │
│   ★ = 关键节点 (金)            │  [最短路径 TIME]   │
│   ╳ = 关键边 (紫虚)           │  [MST 生成树]      │
│                                │  [关键节点/边]     │
│                                │  [清除高亮]        │
│                                │                     │
│                                │  🧪 测试           │
│                                │  [运行 testall]    │
│                                │  (下方显示结果)    │
├────────────────────────────────┴─────────────────────┤
│  状态栏: 顶点 87 · 边 26 · 连通分量 1                 │
└──────────────────────────────────────────────────────┘
```

### 4.2 交互流程

1. **加载数据**：下拉选数据集 → 点"加载" → 地图显示所有节点和边
2. **选起点/终点**：点击地图节点（或右侧输入框），选中的节点高亮
3. **执行算法**：点按钮 → 结果高亮在地图上
4. **悬停节点**：tooltip 显示 `place_id + display_name + category`
5. **拖拽节点**：按住拖动调整位置
6. **testall**：点击后运行全部测试，结果在控制面板下方滚动显示

### 4.3 视觉规范

| 元素 | 样式 |
|------|------|
| 节点 | 圆点半径 6，按 category 着色（Teaching=蓝, Dining=橙, Dormitory=绿, Sports=红, Medical=白, Other=灰） |
| open 边 | 绿色细线 `#4CAF50` |
| closed 边 | 红色虚线 `#F44336` |
| 最短路径 | 蓝色粗线 `#2196F3`，宽度 4 |
| MST 边 | 橙色线 `#FF9800` |
| 关键节点 | 金色外圈 `#FFD700`，半径 +4 |
| 关键边 | 紫色虚线 `#9C27B0` |
| 选中节点 | 白色外圈，半径 +2 |

### 4.4 亮点（评分 1 分）

1. **悬停 tooltip**：鼠标悬停显示地点名称、类别、时间
2. **类别着色**：6 种 category 不同颜色
3. **拖拽布局**：节点可拖动重新定位（Leaflet `draggable: true`，改动 <5 行）
