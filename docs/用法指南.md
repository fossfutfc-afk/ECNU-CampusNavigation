## 环境准备

你需要以下工具：

| 工具 | 检查命令 | 下载地址 |
|------|----------|----------|
| **g++** (C++ 编译器) | `g++ --version` | [TDM-GCC](https://jmeubank.github.io/tdm-gcc/) 或 [MinGW-w64](https://www.mingw-w64.org/) |
| **CMake** (构建系统，可选) | `cmake --version` | [cmake.org](https://cmake.org/download/) |

> **Windows 用户注意**：安装 MinGW 后，需要把 `bin` 目录（如 `C:/TDM-GCC-64/bin`）加入系统环境变量 `PATH`。

## 第一步：编译项目

打开终端，进入项目目录：

```bash
cd "d:/CampusNavigation/Project"
```

### 方式 A：g++ 直接编译（推荐）

**主程序：**

```bash
g++ -std=c++17 -O2 -o build/CampusNavigation.exe src/*.cpp -I include
```

**测试框架：**

```bash
g++ -std=c++17 -O2 -o test/testall.exe test/testall.cpp
```

### 方式 B：CMake

> **重要**：Windows 上 CMake 默认选择 Visual Studio（MSVC），必须用 `-G "MinGW Makefiles"` 指定 MinGW 的 g++。CMakeLists.txt 已内置 `-O2` 优化，无需额外指定。

```bash
mkdir build
cd build
cmake .. -G "MinGW Makefiles"
cmake --build .
```

> 如果 `mkdir` 报目录已存在，忽略即可。产物和方式 A 相同，都在 `build/` 下：
> - `build/CampusNavigation.exe`
> - `build/testall.exe`

### 4.4 状态说明

| 状态 | 颜色 | 含义 |
|------|------|------|
| ` AC ` | 🟢 绿色 | 输出与参考答案**完全一致** |
| ` SEM` | 🟡 黄色 | 输出**不一致但语义等价** — 如 MST 等权边选了不同的合法方案、最短路径等代价不同路线。算法正确，tie-breaking 差异。 |
| ` WA ` | 🔴 红色 | **真实差异** — 输出错误，需排查代码 |
| `CRASH` | 🔴 红色 | 程序运行异常或崩溃 |

### 4.5 语义等价的判断规则

测试框架对以下输出进行语义级对比（而非字符级）：

- **PATH 行**：比较 `模式(DIST/TIME)`、`总代价`、`起点`、`终点`——只要这四个一致，路径中间节点不同也算语义等价（Dijkstra 在等权边时的不同选择）
- **MST 行**：比较 `总距离`、`边数`——只要 MST 总权和边数一致，边的排列不同也算语义等价（Prim/Kruskal 在等权边时的不同选择）
- **其他行**（PLACE, ADJ, CATEGORY, COMPONENTS, CRITICAL, ERROR 等）：必须字符级完全一致

## 第五步：Web 图形化界面（拓展任务 3）

> 此 Web 界面由我提供设计思路和交互需求，由 AI（DeepSeek V4 Pro，通过 Claude Code 工具调用）给自己撰写prompt并完成全部代码编写。我负责需求定义、功能验收和调试反馈，AI 负责 bridge.py、index.html 的具体实现。

项目附带一个基于浏览器 + Flask 中间层的图形化界面，无需修改 C++ 代码。

### 5.1 环境准备

本图形化界面依赖以下环境，请在启动前确认：

| 依赖项 | 说明 |
|--------|------|
| **Python 3.13+** | 用于运行中间层 `bridge.py` |
| **Flask 库** | 首次使用需执行 `pip install flask` 安装 |
| **现代浏览器** | 推荐 Chrome / Edge / Firefox 最新版 |

### 5.2 启动

```bash
cd D:\CampusNavigation\Project\web
pip install flask        # 首次需安装
python bridge.py
```

浏览器打开 `http://localhost:8080`。

### 5.3 功能

| 操作 | 方式 |
|------|------|
| 加载数据 | 下拉选择数据集 → 自动渲染 |
| 选起/终点 | 点击地图节点（再次点击切换，✕ 清除） |
| SHORTEST | DIST / TIME 按钮，路径蓝粗线高亮 |
| TIMED_SHORTEST | 输入时刻 HH:MM → 点 TIMED |
| SHORTEST_K | 输入 K → 点 SHORTEST_K，用券边标黄色虚线 |
| MUST_PASS | 输入必经点（空格分隔）→ 点 MUST_PASS，粉色路径 |
| MST | 橙色标出生成树边 |
| CRITICAL | 割点金色边框 + 桥紫色虚线（内置 Tarjan O(V+E)） |
| COMPONENTS | 弹窗显示连通分量数和规模 |
| 悬停节点 | 显示 place_id、display_name、category、开放时间 |
| 拖拽节点 | 按住拖动可重新布局 |
| 类别着色 | Teaching=蓝, Dining=橙, Dormitory=绿, Sports=红, Medical=黄 |
| 非法数据 | 深红虚线标出，警告横幅，算法按钮禁用 |

### 5.4 架构

```
浏览器 (Leaflet.js)  ←→  bridge.py (Flask)  ←→  CampusNavigation.exe
```

Flask 中间层将用户操作翻译为 CLI 命令发给 exe，解析输出返回 JSON。前端用 Leaflet.js + L.CRS.Simple 在自定义坐标系上渲染节点和边。

## 命令速查

### 数据加载与保存

| 命令 | 格式 | 成功输出 |
|------|------|----------|
| LOAD | `LOAD <places.csv> <roads.csv>` | `OK` |
| SAVE | `SAVE <places.csv> <roads.csv>` | `OK` |

### 查询

| 命令 | 格式 | 输出示例 |
|------|------|----------|
| QUERY_PLACE | `QUERY_PLACE <place_id>` | `PLACE P0001 Library Teaching 30 08:00 22:00` |
| QUERY_CATEGORY | `QUERY_CATEGORY <category>` | `CATEGORY Teaching 2 P0001 P0005` |
| ADJ | `ADJ <place_id>` | `ADJ P0001 3 P0002:200:3:open ...` |

### 数据维护

| 命令 | 格式 |
|------|------|
| ADD_PLACE | `ADD_PLACE <id> <name> <cat> <stay> <open> <close>` |
| DELETE_PLACE | `DELETE_PLACE <place_id>` |
| UPDATE_PLACE | `UPDATE_PLACE <place_id> <field> <value>` |
| ADD_ROAD | `ADD_ROAD <from> <to> <dist> <time> <status>` |
| DELETE_ROAD | `DELETE_ROAD <from> <to>` |
| UPDATE_ROAD | `UPDATE_ROAD <from> <to> <field> <value>` |
| CLOSE_ROAD | `CLOSE_ROAD <from> <to>` |
| OPEN_ROAD | `OPEN_ROAD <from> <to>` |

### 图算法（必做）

| 命令 | 格式 | 输出示例 |
|------|------|----------|
| COMPONENTS | `COMPONENTS` | `COMPONENTS 1 SIZES 6` |
| SHORTEST | `SHORTEST <from> <to> <DIST\|TIME>` | `PATH DIST 680 NODES P0001 P0003 P0005 P0006` |
| TIMED_SHORTEST | `TIMED_SHORTEST <from> <to> <HH:MM> <DIST\|TIME>` | `PATH TIME 9 NODES ...` |
| MUST_PASS | `MUST_PASS <from> <to> <DIST\|TIME> <k> <p1> ... <pk>` | 同上 |
| MST | `MST` | `MST 980 EDGES P0001-P0002:200 ...` |
| CRITICAL | `CRITICAL` | `CRITICAL NODES 1 P0005 EDGES 1 P0005-P0006` |

### 图算法（拓展 / 加分）

| 命令 | 格式 | 说明 |
|------|------|------|
| SHORTEST_K | `SHORTEST_K <from> <to> <K>` | 共享单车：K 张加速券，输出 `PATH <time> K_USED <n> NODES ... FAST <k> [<u>-<v> ...]` |
| CRITICAL_TARJAN | `CRITICAL_TARJAN` | Tarjan O(V+E) 版割点/桥，输出格式与 CRITICAL 相同 |

### 其他

| 命令 | 作用 |
|------|------|
| `QUIT` | 退出程序 |

### 常见错误输出

| 输出 | 含义 |
|------|------|
| `ERROR place_not_found` | 地点不存在 |
| `ERROR place_already_exists` | 地点已存在 |
| `ERROR road_not_found` | 道路不存在 |
| `ERROR road_already_exists` | 道路已存在 |
| `ERROR invalid_field` | 不支持的字段名 |
| `NO_PATH` | 两点不可达 |
| `DISCONNECTED` | 图不连通（MST 无法生成） |
